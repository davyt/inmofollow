Configuración y gestión

Todo lo que el administrador necesita saber para poner en marcha y mantener InmoFollow.

Checklist de puesta en marcha

Seguí este orden para dejar el sistema listo para los agentes:

  • Completar los datos básicos de la empresa en Configuración → Mi empresa (nombre, email, teléfono)
  • Completar las credenciales de WhatsApp en Configuración → WhatsApp / Meta (Phone Number ID y Access Token)
  • Agregar el WhatsApp Business Account ID (WABA ID) — necesario para ver plantillas aprobadas
  • Probar la conexión con el botón "Probar conexión" en la página de WhatsApp / Meta
  • Crear los estados del Pipeline (ej: Nuevo, En seguimiento, Calificado, Cerrado) — desde el Pipeline directamente
  • Crear las plantillas de mensajes globales (primer contacto, seguimiento, etc.) — con IA o manual
  • Crear los flows de automatización y asociarlos a triggers (cambio de estado o lead creado), con filtro de origen si corresponde
  • Crear los usuarios (agentes y supervisores)
  • Configurar el Agente IA en Configuración → Agentes IA — opcional pero recomendado
  • Importar los leads iniciales o cargarlos manualmente
  • Asignar los leads a los agentes
  • Verificar que el cron job esté activo en el servidor (php artisan schedule:run cada minuto)

Configurar WhatsApp Business desde cero

Si todavía no tenés la API de WhatsApp configurada, seguí estos pasos:

⚠️

Este proceso requiere una cuenta de Meta Business Manager verificada y un número de teléfono que no esté siendo usado en la app de WhatsApp. Si el número ya tiene WhatsApp instalado, hay que desvincularlo primero.

1

Crear o acceder a Meta Business Manager

Entrá a Meta Business Manager con tu cuenta de Facebook.

↗ business.facebook.com
2

Crear una aplicación en Meta Developers

Meta Developers → Mis Apps → Crear app. Tipo "Empresa", completá el nombre.

↗ developers.facebook.com/apps
3

Agregar el producto WhatsApp a la app

Dentro de la app, buscá el producto WhatsApp en el panel de productos y hacé clic en Configurar.

4

Agregar y verificar el número de teléfono

En WhatsApp → Configuración → Números de teléfono, agregá el número. Meta enviará un código de verificación por SMS o llamada.

5

Obtener el Phone Number ID y WABA ID

Una vez verificado, en el mismo panel verás el Phone Number ID. En Meta Business Manager → Configuración del negocio → Cuentas de WhatsApp encontrás el WABA ID.

6

Obtener el Access Token (System User recomendado)

El token temporal que muestra Meta Developers vence pronto. Para producción, creá un System User en Meta Business Manager con permisos de WhatsApp y generá un token permanente.

Ruta: Meta Business Manager → Configuración del negocio → Usuarios del sistema → Agregar → Asignale el activo de WhatsApp con rol de administrador → Generar token.

↗ Usuarios del sistema
7

Configurar el Webhook

Para que el sistema detecte respuestas entrantes, configurá el webhook en Meta Developers → Webhooks:

URL: https://tudominio.com/webhooks/whatsapp
Identificador de verificación: valor de WHATSAPP_VERIFY_TOKEN en el .env del servidor (por defecto inmofollow_wh_token).
Suscribite al campo messages.

Paso que se suele olvidar: andá a WhatsApp → Introducción → Configuración de producción, buscá tu número y activá "Suscribirse a webhooks". Sin este paso, el webhook nunca recibe mensajes aunque la configuración de arriba esté perfecta.

8

Cargar las credenciales en InmoFollow

Ingresá el Phone Number ID, el Access Token y el WABA ID en Configuración → WhatsApp / Meta y guardá. Luego usá "Probar conexión" para confirmar.

Plantillas aprobadas por Meta

Para contactar leads que nunca te escribieron (o que pasaron más de 24hs sin responder), WhatsApp exige usar una plantilla aprobada previamente por Meta.

1

Ir a Plantillas de mensajes en Meta

↗ Meta → Plantillas de mensajes
2

Crear nueva plantilla

Categoría Marketing, nombre en minúsculas sin espacios (ej: primer_contacto_ml), idioma español.

3

Escribir el mensaje con variables

Usá {{1}}, {{2}}, etc. donde van datos del lead. Ejemplo: "Hola {{1}}, vimos tu publicación en MercadoLibre sobre la propiedad en {{2}}. ¿Podemos charlar?"

4

Esperar aprobación (24-48hs)

Meta revisa la plantilla. El estado cambia a Aprobada. Si la rechazan, ajustá el texto y reenviá.

5

Conectarla a una plantilla en InmoFollow

Andá a Configuración → Plantillas, editá la plantilla y completá la sección "Primer contacto por WhatsApp": nombre de la plantilla en Meta (idéntico), idioma y variables en orden.

✅

Desde Configuración → WhatsApp / Meta → "Ver plantillas aprobadas" podés ver en tiempo real qué plantillas están aprobadas, pendientes o rechazadas en tu cuenta de Meta.

Gestión de usuarios

Accedé desde Configuración → Usuarios. Podés crear, editar y desactivar usuarios.

Crear un agente

  1. Clic en "Nuevo usuario"
  2. Completá nombre, email y contraseña inicial
  3. Seleccioná rol: Agente o Supervisor
  4. Guardá y compartí las credenciales

Cambiar contraseña de un agente

Desde la tabla de usuarios hacé clic en "Nueva contraseña" junto a su nombre. El agente puede usarla de inmediato.

Plantillas de mensajes

Las plantillas globales están disponibles para todos los agentes. Las personales solo las ve el agente que las creó. Como administrador, solo vos podés crear y editar plantillas globales (accedé desde Configuración → Plantillas).

Pipeline comercial

El Pipeline es la vista Kanban del proceso de ventas. Los leads se arrastran entre columnas para cambiar su estado. Accedé desde el menú Comercial → Pipeline.

Gestionar los estados

Hacé clic en el ícono ⚙ del encabezado de cualquier columna para editar ese estado: renombrar, cambiar color, reordenar con las flechas ← →, o eliminar (los leads quedan sin estado, sin perder datos). Usá "+ Añadir estado" al final del tablero para crear uno nuevo.

💡

Cuando movés un lead de columna (drag & drop), el sistema dispara automáticamente el flow asociado a ese nuevo estado si existe uno activo y compatible con el origen del lead.

Flows de automatización

Los flows son secuencias de acciones que se ejecutan automáticamente cuando ocurre un evento. Accedé desde Automatizaciones → Flows.

Tipos de trigger (disparador)

TriggerCuándo se dispara
Cambio de estadoCuando el lead pasa al estado seleccionado (por drag & drop en Pipeline, cambio manual, o acción de IA)
Lead creadoCada vez que se crea un lead nuevo (manual, importación CSV)

Filtro por origen del lead

Al crear un flow, el campo "Solo si Origen es" permite que el flow solo se dispare cuando el lead viene de una fuente específica:

  • ml — solo leads de MercadoLibre
  • 2clics — solo leads importados de 2clics
  • web / manual — otros orígenes
  • (vacío) — se dispara para cualquier origen

Esto permite tener un flow de bienvenida diferente para leads de MercadoLibre (que ya tenés su propiedad) versus leads genéricos.

Tipos de pasos disponibles

TipoQué haceCuándo usarlo
📋 Enviar plantillaEnvía una plantilla aprobada por WhatsApp o email al número de días indicadoPrimer contacto, seguimientos programados
💬 Mensaje libreEnvía texto libre por WhatsApp (solo funciona si hay sesión activa de 24hs)Confirmaciones rápidas dentro de una conversación activa

Crear un flow

  1. Ir a Automatizaciones → Flows → "+ Nuevo"
  2. Elegir tipo de trigger (cambio de estado / lead creado), el estado si aplica, y el origen si querés filtrarlo
  3. Crear el flow
  4. Agregar pasos con el tipo, datos y número de días
  5. Activar el flow con el botón "▶ Activar"
💡

Los pasos con Día 0 se ejecutan de inmediato. Las acciones (cambiar estado, asignar agente) en Día 0 se aplican en el momento, sin pasar por la cola de mensajes.

⚠️

El primer paso de un flow para leads nuevos debería ser Enviar plantilla (aprobada por Meta), ya que el lead no habrá iniciado conversación. Un "Mensaje libre" en Día 0 no puede enviarse sin sesión activa de 24hs.

Broadcasts (envíos masivos)

Los broadcasts permiten enviar una plantilla aprobada por Meta a un conjunto de leads a la vez. Accedé desde Comunicación → Broadcast.

  1. Elegir la plantilla (debe estar aprobada por Meta)
  2. Seleccionar los leads destinatarios (filtros por estado, agente, zona, etc.)
  3. Confirmar el envío
⚠️

Solo se envía a leads con consentimiento de WhatsApp activo. Los mensajes se procesan a través del mismo cron que los mensajes de flows, respetando el límite de 20 por corrida.

Importar leads

Desde Comercial → Leads → Importar podés subir un archivo CSV o Excel con leads masivos. El proceso tiene 3 pasos.

Paso 1: Subir el archivo

El sistema acepta CSV (.csv, .txt) y Excel (.xlsx). Detecta la fila de cabecera automáticamente.

Paso 2: Mapear columnas

Encontrás tres elementos en este paso:

  • Botón "⚡ Preset MercadoLibre" — si el archivo viene del export de ML, este botón asigna automáticamente las 9 columnas (Item ID, Teléfono, Título, Tipo, Operación, Zona, Precio, Atributos, Link). También fija el origen en ml.
  • Origen fijo — etiqueta a todos los leads del archivo con ese origen. Usá valores cortos: ml, 2clics, web. Este valor activa los flows que filtrán por origen.
  • Perfil guardado — podés guardar el mapeo actual con un nombre para reutilizarlo en futuras importaciones del mismo tipo.

📋 Sección: Datos del contacto

Columnas que van al perfil del lead: nombre (opcional si hay propiedad), teléfono, email, zona, tipo de propiedad, origen, estado, agente, consentimientos, notas.

🏠 Sección: Datos de la propiedad

Columnas que van al expediente de propiedad del lead. Si dejás todo vacío, no se crea expediente. Campos disponibles:

  • ID externo — Item ID de MercadoLibre. Sirve para no duplicar al reimportar.
  • Título — título de la publicación
  • Tipo de propiedad — (apartamento, casa, etc.)
  • Operación — venta o alquiler
  • Precio — acepta el formato del CSV de ML: "145000 dólares" o "680000 pesos"
  • Zona descriptiva — texto libre de la zona
  • Link publicación — URL de la publicación en ML
  • Atributos — acepta el formato: "4 dormitorios | 2 baños | 174 m² cubiertos"

Paso 3: Resultado

Muestra 4 contadores: Leads nuevos (creados), Ya existían (duplicados por teléfono — se les agregó la propiedad), Omitidos y Errores. Si hay estados que no coincidieron con ninguno del pipeline, se listan por separado.

💡

Si un lead con el mismo teléfono ya existe, no se duplica — se agrega la propiedad al lead existente y se cuenta en "Ya existían". Al reimportar el mismo CSV de ML, las propiedades ya importadas tampoco se duplican (se actualizan por el Item ID).

⚠️

Antes de importar, asegurate de que la columna de consentimiento de WhatsApp esté mapeada si los leads ya dieron su ok. Sin consentimiento, el sistema no les envía mensajes automáticos.

Asignar leads a agentes

  • Individual: editá el lead y seleccioná el agente en el campo "Agente".
  • Masivo: en la tabla de leads, seleccioná varios con el checkbox → Acciones → Asignar agente.

System prompt

Define el comportamiento del agente: nombre, tono, qué puede y no puede responder. Ejemplo:

"Sos Valentina, asistente virtual de Inmobiliaria Rivera. Respondé en español rioplatense, de forma cálida y profesional. Solo respondé consultas sobre propiedades en venta o alquiler. Si el cliente quiere agendar una visita, decile que un agente lo contactará en el día."

Modo de operación

ModoQué pasa
Auto-send activadoEl agente genera y envía la respuesta automáticamente al cliente
Auto-send desactivadoEl agente genera un borrador que aparece en el Inbox para que el agente humano lo apruebe o descarte

Base de conocimiento

Agregá contexto específico para que el agente responda con información real:

  • Texto — descripción de servicios, zonas, horarios, precios
  • URL — el sistema visita la página y extrae el contenido automáticamente
  • PDF — subí un documento (tarifario, reglamento, listado) y el sistema extrae el texto

Cada entrada se puede activar o desactivar sin eliminarla. El agente usa hasta 6.000 caracteres de la base de conocimiento por respuesta.

Contexto de propiedad automático

Si el lead tiene un expediente de propiedad primario (importado desde ML), el agente lo recibe automáticamente en cada conversación: operación, precio publicado, dormitorios, m² cubiertos y link. No requiere configuración adicional.

Pestaña Playground

Permite probar el agente con mensajes de prueba antes de activarlo. Podés escribir una consulta como si fueras un lead y ver exactamente qué responde el agente con la configuración actual. Ideal para ajustar el system prompt y la base de conocimiento antes de activar el auto-send.

💰

El costo de la IA se descuenta de la cuenta del proveedor que configuraste. Para generación de textos de plantillas, el consumo es muy bajo. Para respuestas automáticas, depende del volumen de mensajes.
